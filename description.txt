Формат выходного файла при кодировании (алгоритм Хаффмана)

1. 4 байта: int uniqueCount – количество уникальных символов во входном файле.
   Если uniqueCount == 0, файл пуст, дальнейшие поля отсутствуют.

2. Для каждого уникального символа (повторяется uniqueCount раз):
   - 1 байт: byte symbol – сам символ (0..255).
   - 4 байта: int frequency – частота встречаемости.

3. 1 байт: byte validBits – количество значащих битов в последнем байте закодированных данных.
   Если validBits == 0, то закодированные данные отсутствуют (случай одного уникального символа или пустого файла).

4. Далее: байтовый массив encodedData – битовый поток, упакованный в байты.
   - Байты идут строго по порядку.
   - В каждом байте биты расположены от старшего (bit 7) к младшему (bit 0).
   - Последний байт содержит ровно validBits значащих битов (остальные биты – заполнители нулями).
   - Если validBits == 0, то этот массив пуст.

Словарь (таблица кодов) явно не хранится, вместо него сохраняются частоты символов.
При декодировании по частотам строится идентичное дерево Хаффмана.
